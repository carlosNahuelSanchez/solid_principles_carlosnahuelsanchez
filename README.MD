# Principios SOLID - Ejemplos en TypeScript

Este repositorio contiene ejemplos prácticos de los cinco principios de diseño de software SOLID, implementados en TypeScript.

## ¿Qué son los principios SOLID?

SOLID es un acrónimo mnemotécnico para cinco principios de diseño de programación orientada a objetos y diseño de software. Estos principios, cuando se aplican juntos, intentan hacer que los diseños de software sean más comprensibles, flexibles y mantenibles.

- **S** - Single Responsibility Principle (Principio de Responsabilidad Única)
- **O** - Open/Closed Principle (Principio de Abierto/Cerrado)
- **L** - Liskov Substitution Principle (Principio de Sustitución de Liskov)
- **I** - Interface Segregation Principle (Principio de Segregación de Interfaces)
- **D** - Dependency Inversion Principle (Principio de Inversión de Dependencias)

## ¿Cómo ejecutar los ejemplos?

Primero, instala las dependencias del proyecto:

```bash
npm install
```

### Ejecutar todos los principios a la vez

Para ver todos los ejemplos de los principios SOLID en acción, ejecuta el siguiente comando:

```bash
npm start
```

### Ejecutar cada principio individualmente

Si deseas ejecutar un principio específico, puedes usar los siguientes comandos:

- **Principio de Responsabilidad Única (SRP):**
  ```bash
  npm run srp
  ```

- **Principio de Abierto/Cerrado (OCP):**
  ```bash
  npm run ocp
  ```

- **Principio de Sustitución de Liskov (LSP):**
  ```bash
  npm run lsp
  ```

- **Principio de Segregación de Interfaces (ISP):**
  ```bash
  npm run isp
  ```

- **Principio de Inversión de Dependencias (DIP):**
  ```bash
  npm run dip
  ```
